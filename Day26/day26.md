# 🏥 Day 26 – Prior Authorization Workflow Simulator

## 📌 Overview

The **Prior Authorization Workflow Simulator** is an interactive web application that demonstrates the complete healthcare prior authorization (PA) process. It allows users to simulate real-world patient scenarios, move cases through different workflow stages, collect required documentation, submit requests to payers, and experience approval, pend, denial, and appeal outcomes.

This project was developed as part of the **Day 26 AI Challenge** using a single self-contained HTML file. :contentReference[oaicite:0]{index=0}

## ✨ Features

- 🏥 Multiple Patient Scenarios
- 🩺 Medical Necessity Evaluation
- 📂 Prior Authorization Document Collection
- 📤 Submit Requests to Payer
- 🏦 Payer Review Simulation
- ✅ Approval Workflow
- ⏸️ Pend Workflow
- ❌ Denial & Appeal Workflow
- 📞 Peer-to-Peer Review
- 📅 Journey Tracker
- ⏳ Elapsed Days Counter
- 📊 Workflow Summary Dashboard
- 🎉 Confetti Animation on Approval
- 📱 Fully Responsive UI
- 🌙 Modern Healthcare Dashboard Design

---

# 🖥️ Demo Workflow

The simulator guides users through:

1. Select a Patient Scenario
2. Review Medical Necessity
3. Collect Required Documents
4. Submit Prior Authorization
5. Payer Review
6. Approval / Pend / Denial
7. Appeal or Peer-to-Peer Review (if required)
8. Workflow Summary
9. Restart with Another Patient
