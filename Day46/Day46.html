Workflow — a real cycle, not a straight pipeline

Architecture overview

      

        This pipeline runs as a real, unbounded loop against the Claude API: Safety Monitor gates the request once, Planner and
        Executor produce an initial draft, then Evaluator → Critic → Improver repeat as separate live API calls each round.
        A Memory Manager call threads persistent lessons forward so the Improver doesn't regress fixes from earlier rounds.
        After every round the stop-check runs in a fixed order — plateau, then threshold, then hard cap — and only when it fires
        does the branch to Final Reviewer activate. Nothing about the round count is hardcoded: it is a runtime result of that check.
      

    


    

      
Extension ideas

      
    
        
  • Add a Test Writer agent that generates unit tests alongside the Executor, and feed pass/fail results into the Evaluator's correctness score.
    • 
        
  • Add a second Safety Monitor pass after each Improver round, so revisions can't drift into unsafe territory.
    • 
        
  • Let the Critic vote on whether an issue is "blocking" vs "nice-to-have" and weight the Evaluator's next-round rubric accordingly.
    • 
        
  • Persist the Memory Manager's digest across sessions (e.g. via local storage or a backend) so lessons carry over between unrelated runs.
    • 
        
  • Add a human-in-the-loop checkpoint: pause after the plateau safety net fires and let a person approve continuing past the hard cap.
    • 
        
  • Swap the domain: point the same loop at marketing copy (Evaluator scores clarity/persuasion/brand-fit) or research synthesis (Evaluator scores citation coverage/accuracy).

Further prompts for building more advanced autonomous systems

      
"Add a Test Writer agent to this pipeline that generates and mentally executes unit tests, and wire its pass/fail output into the Evaluator's correctness dimension."

      
"Introduce a branch where, if the Safety Monitor flags a concern short of a full block, a Human Review agent summarizes the risk instead of auto-continuing."

      
"Give the Memory Manager long-term persistence across runs, and show a diff of what it learned this run vs. what it already knew."

      
"Parallelize the loop: run two Improver variants per round, have the Evaluator score both, and keep only the higher-scoring branch (best-of-n)."

      
"Retarget this studio at a different workflow — e.g. a research-report pipeline with Planner, Researcher, Fact-Checker, Synthesizer, and Editor agents — reusing the same loop/stop-check architecture."
