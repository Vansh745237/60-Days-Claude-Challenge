The system prompt

The assistant runs on a single Claude call per submission, constrained to one job: turn a student's raw background into a rewritten LinkedIn headline, About section, and experience bullets, plus a short coach critique. The prompt fixes role, scope, tone, and a strict JSON output contract so the UI can render it into the mock profile card without any text-parsing guesswork.



      
Why the UI looks like this

      
A generic chat box would hide the actual deliverable behind scrollback. Students revising a profile need to see the rewrite sitting where it will actually live — so the output renders inside a mock LinkedIn card (banner, avatar circle, headline, About, experience) rather than as a chat bubble. The red-pen "coach's markup" panel mirrors how a mentor actually reviews a draft: strengths, fixes, and tips tagged and separated, in the blunt-coach voice the user asked for, not softened chatbot hedging.

      
The score strip (Headline strength, About clarity, Keyword alignment) gives an at-a-glance verdict before the person reads a single rewritten sentence — useful for early-career users who don't yet know what "good" looks like on LinkedIn.


      
Extending this

      
    
        
  • Add a job-description matcher: accept a pasted JD as a second input and have the model weight keyword alignment against that specific posting instead of the target role in general.
    • 
        
  • Add memory: persist the last submission in localStorage so returning users see their previous draft and a diff against the new one — turns a one-shot tool into a revision tracker.
    • 
        
  • Multi-step flow: split into two calls — first ask clarifying questions if the achievements are too thin (e.g. "What was the measurable outcome of the hackathon project?"), then generate the rewrite once the gaps are filled.
    • 
        
  • Add tools: give Claude a web_search tool to pull current phrasing conventions for a specific industry (e.g. how PM postings phrase impact) before writing the headline, so the rewrite reflects live norms rather than only the model's training data.
    • 
        
  • Section-by-section regeneration: let users click "regenerate" on just the headline or just one bullet, sending a smaller follow-up call with the rest of the profile as context instead of re-running the whole brief.
    • 
      


      
Edge cases the prompt handles

      
    
        
  • Missing or too-thin achievements → the model is instructed to ask for one concrete detail rather than inventing accomplishments.
    • 
        
  • Irrelevant or nonsense input → the model declines to fabricate a profile and returns a short redirect message in the same JSON contract.
    • 
        
  • Abusive or off-topic input → the model stays in scope, does not engage, and returns a neutral refusal note instead of a profile.
